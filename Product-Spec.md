# 产品需求文档 (Product Specification)

## 项目概述

**项目名称**：双人课程表

**一句话描述**：一款面向情侣的双人课程表应用，可同时显示两个人的当前课程状态，方便了解对方日程安排。

**目标**：帮助情侣实时了解对方的上课状态，便于安排约会和联系时间。

## 目标用户

**核心用户群体**：情侣

**用户特征**：
- 年龄段：18-25岁（大学生为主）
- 技术水平：熟练使用智能手机，习惯使用课程表类应用
- 使用场景：想了解对方当前是否在上课，方便约会、联系、规划共同时间
- 痛点：无法快速知道对方是否在上课，需要反复询问；难以找到双方共同的空闲时间

---

## 核心功能

### 功能 1：双人课程表管理

**功能描述**：支持在本地存储和管理两个人的独立课程表，用户可以分别编辑两个人的课程信息。

**输入**：
- 课程名称：字符串，如"高等数学"
- 上课地点：字符串，如"教学楼A101"
- 上课时间：课节选择，如"第1-2节"
- 周次设置：支持单双周、多周循环，如"1-16周"、"单周"、"双周"
- 星期：1-7，表示周一到周日
- 所属人：标识课程属于哪个人

**输出**：
- 课程列表：按时间排序的课程数据
- 课程详情：单门课程的完整信息

**业务规则**：
- 两个课程表相互独立，互不影响
- 支持复杂周期设置：单周、双周、1-16周任意组合
- 课程时间不能重叠（同一人的课程）
- 本地存储，无需网络
- 周次选择使用点选框，根据学期总周数动态显示所有周数
- 点击周数表示该周上课，再次点击取消选择
- 支持滑动选择多周，再次滑动可取消选择
- 周数显示格式：连续周数用横杠连接（如"1-5周"），不连续周数用逗号分隔（如"1,3,5周"），混合情况用逗号分隔（如"1-5,8,10周"），单周显示"第X周"

**异常处理**：
- 时间冲突：提示用户时间重叠，不允许保存
- 数据为空：课程名称为必填项，其他可选

**优先级**：高

**状态**：已实现

### 功能 2：当前课程状态显示

**功能描述**：在主页实时显示两个人当前正在上的课程，包括课程名称和剩余时间。

**输入**：
- 当前系统时间
- 两个人的课程表数据

**输出**：
- 人员A当前课程：课程名称 + 剩余时间
- 人员B当前课程：课程名称 + 剩余时间
- 无课状态：显示"无课"

**业务规则**：
- 根据当前时间自动匹配课程
- 需要考虑当前周次（单周/双周）
- 剩余时间实时更新（分钟级）
- 当前无课时显示"无课"

**异常处理**：
- 课程表为空：显示"请添加课程"
- 时间获取失败：使用上次成功获取的时间

**优先级**：高

**状态**：已实现

### 功能 4：共同空闲时间

**功能描述**：计算并显示两个人今天共同的空闲时间段，方便安排约会。

**输入**：
- 两个人的今日课程列表
- 当前时间

**输出**：
- 共同空闲时间段列表：如"12:00-14:00"、"18:00之后"

**业务规则**：

- 点击空闲时间卡片可查看所有空闲时间列表
- 点击列表中的单个时段可查看详情

**异常处理**：
- 无共同空闲时间：显示"今天没有共同空闲时间"

**优先级**：中

**状态**：已实现

### 功能 5：桌面小组件

**功能描述**：提供多种尺寸的桌面小组件，无需打开应用即可查看课程信息。支持MIUI/HyperOS负一屏（桌面助手）添加。为两个人分别提供独立的个人小组件。

**输入**：
- 两个人的课程表数据
- 当前系统时间
- 当前周次信息

**输出**：
- 2×2 小组件：共同空闲时间
- 4×2 小组件：我的今日课程 / Ta的今日课程（两个独立组件）

**业务规则**：

**1. 小组件尺寸规格**

| 尺寸 | 名称 | 显示内容 |
|------|------|----------|
| 2×2 | 空闲时间小组件 | 显示两人当前共同的空闲时间 |
| 4×2 | 我的今日课程小组件 | 显示"我"的今日课程列表 |
| 4×2 | Ta的今日课程小组件 | 显示"Ta"的今日课程列表 |

**2. 2×2 空闲时间小组件**
- 显示当前最近的共同空闲时间段
- 格式：如"12:00-14:00 空闲"
- 无空闲时间时显示"今日无共同空闲"
- 点击打开应用主页

**3. 4×2 我的今日课程小组件**
- 仅显示"我"（PersonB）的今日课程列表
- 每门课程显示：课程名称、上课地点、节次
- 已结束课程灰色显示
- 使用黄色主题色标识
- 点击打开"我的课表"页面

**4. 4×2 Ta的今日课程小组件**
- 仅显示"Ta"（PersonA）的今日课程列表
- 每门课程显示：课程名称、上课地点、节次
- 已结束课程灰色显示
- 使用蓝色主题色标识
- 点击打开"Ta的课表"页面

**5. 视觉风格**
- 采用 iOS 风格设计
- 圆角卡片样式（16dp 圆角）
- 浅色/深色模式自适应
- 毛玻璃背景效果（可选）
- 个人小组件通过颜色区分（我：黄色，Ta：蓝色）

**6. 刷新机制**
- 智能刷新：课程状态变化时自动刷新
- 屏幕亮起时刷新
- 最小刷新间隔：1分钟

**7. MIUI/HyperOS负一屏支持**
- 支持小米桌面助手/负一屏添加小组件
- 在负一屏小组件列表中可见所有小组件
- MIUI小组件配置：
  - miuiWidget: true
  - miuiWidgetRefresh: exposure
  - miuiWidgetRefreshMinInterval: 60000
  - vivo_widget: true
  - vivoWidgetVersion: 1

**8. 点击交互**
- 空闲时间小组件：打开应用主页
- 我的今日课程小组件：打开"我的课表"页面
- Ta的今日课程小组件：打开"Ta的课表"页面

**异常处理**：
- 数据读取失败：显示"加载中..."
- 无课程数据：显示"暂无课程"

**优先级**：高

**状态**：已实现

### 功能 6：课程数据导入

**功能描述**：支持从多种来源导入课程数据，减少手动输入工作量。

**输入**：
- 本应用导出的CSV文件
- Wakeup课程表备份文件
- 凌展教务系统账号密码

**输出**：
- 解析后的课程列表预览
- 导入结果反馈（成功/失败数量）

**业务规则**：

**1. 本应用CSV文件导入**
- 支持导入本应用导出的CSV格式文件
- 支持导入按模板格式填写的CSV文件
- CSV 字段格式（按顺序）：课程名称（必填）、星期（必填）：1-7、开始节次（必填）、结束节次（必填）、教室地点（可选）、上课老师（可选）、周次（可选）
- 导入前显示课程列表预览，用户可勾选要导入的课程
- 用户在预览页面统一选择导入目标：我的课表或Ta的课表
- 冲突处理：用户可选择覆盖现有数据或合并（保留现有数据）
- 冲突课程以红色高亮标记

**2. Wakeup课程表导入**
- 支持从Wakeup课程表应用导出的备份文件导入
- 自动解析Wakeup数据格式，转换为应用课程格式
- 导入前显示课程列表预览
- 用户在预览页面统一选择导入目标：我的课表或Ta的课表

**3. 凌展教务系统导入**
- 用户输入教务系统网址、学号、密码
- 如有验证码，显示验证码图片让用户手动输入
- 应用自动登录教务系统获取课表数据
- 导入前显示课程列表预览，用户可勾选要导入的课程
- 用户在预览页面统一选择导入目标：我的课表或Ta的课表
- 账号密码不保存，每次导入需重新输入
- 导出文件不包含教务系统账号信息

**异常处理**：
- 文件格式错误：提示"文件格式错误，请对照模板检查"
- 部分课程导入失败：显示失败列表和原因
- 导入失败：提供重试选项
- 导入前自动备份现有数据，失败时可恢复
- 教务系统连接失败：提示检查网络或账号密码
- 验证码错误：提示重新输入

**优先级**：高

**状态**：已实现

### 功能 10：课表参数设置

**功能描述**：支持为两个人分别设置课表相关参数，包括开学时间、学期总周数、当前周数、课表节数和课表时间。

**输入**：
- 开学时间：日期选择器，如"2026-02-24"
- 学期总周数：数字输入，如"16"、"18"、"20"
- 当前周数：数字输入，如"第3周"
- 课表节数：数字输入，如"12节"、"14节"
- 课表时间：每节课的开始和结束时间，如"第1节 08:00-08:45"

**输出**：
- 人员A的课表参数配置
- 人员B的课表参数配置

**业务规则**：
- 两人课表参数独立设置，互不影响
- 开学时间用于自动计算当前周数
- 学期总周数限制当前周数的最大值
- 课表节数决定课表网格的行数
- 课表时间用于课表显示和课程时间匹配
- 当前周数可手动调整，也可根据开学时间自动计算

**异常处理**：
- 开学时间晚于当前日期：提示用户检查
- 当前周数超出总周数：自动限制在有效范围内

**优先级**：高

**状态**：已实现

### 功能 16：课表空白格子交互优化

**功能描述**：优化课表页面点击空白格子的交互方式，改为两步操作，避免误触直接进入添加课程界面。

**输入**：
- 点击的格子位置（星期几、第几节）
- 格子是否有课程

**输出**：
- 第一次点击：显示灰色半透明遮罩 + 居中加号图标
- 第二次点击：进入课程添加界面，自动选中对应的星期和节次

**业务规则**：

**1. 空白格子交互（两步操作）**
- 第一次点击空白格子：在该格子上显示灰色半透明遮罩，遮罩中央显示加号图标
- 第二次点击同一格子：进入课程添加界面，自动选中对应的星期和节次
- 点击其他格子：原格子的遮罩消失，新格子显示遮罩
- 点击格子外区域：遮罩消失，恢复原状
- 遮罩不会自动消失，必须用户主动操作

**2. 已有课程格子交互（保持原有行为）**
- 点击有课程的格子：直接进入课程编辑/预览界面
- 不显示遮罩中间状态

**3. 遮罩样式**
- 背景：灰色半透明（深色模式 25% 白色，浅色模式 18% 黑色）
- 图标：白色加号图标，居中显示
- 图标大小：24dp
- 遮罩覆盖整个格子区域
- 无边框

**4. 顶部添加按钮**
- 从顶部添加按钮进入课程编辑页面，不预选任何星期和节次
- 保持原有行为不变

**异常处理**：
- 无

**优先级**：中

**状态**：已实现

- 保持现有的 `navigationBarColor` 为透明设置

**异常处理**：
- Android 10 以下版本忽略此属性，不影响兼容性
- 非小米设备同样受益于透明导航栏效果

- 以该周一作为第 1 周的起始日期
- 后续周数在此基础上正确计算

**技术实现**：
- 修改 `SettingsDataStore.kt` 中的 `getWeekDates` 方法

### 功能 73：iOS 风格页面切换动画

**功能描述**：重构应用的页面切换动画，采用 iOS 风格的滑动切换效果，提升用户体验和视觉流畅度。

**输出**：

- 流畅自然的过渡效果

**业务规则**：

**1. 底部导航栏切换动画**
- 切换到右侧页面：新页面从右侧滑入，当前页面向左滑出
- 切换到左侧页面：新页面从左侧滑入，当前页面向右滑出
- 动画时长：250ms
- 缓动曲线：FastOutSlowInEasing

**2. 页面内部跳转动画（进入二级页面）**
- 新页面从右侧滑入
- 当前页面向左滑出（位移量为屏幕宽度的 30%）
- 动画时长：300ms
- 缓动曲线：FastOutSlowInEasing

**3. 返回动画（返回上一页）**
- 当前页面向右滑出

- 缓动曲线：FastOutSlowInEasing

- 缓动曲线：FastOutSlowInEasing（开始快、结束慢）

**5. 特殊处理**
- 底部导航栏切换时，不应用背景页面位移效果
- 快速连续切换时，动画不打断，平滑过渡到新目标

- 根据导航方向动态设置滑动方向

**2. 动画配置**

```kotlin
    slideInHorizontally(
        animationSpec = tween(300, easing = FastOutSlowInEasing),
        initialOffsetX = { fullWidth -> fullWidth }

        animationSpec = tween(300, easing = FastOutSlowInEasing)
    )
}
exitTransition = {
    slideOutHorizontally(
        animationSpec = tween(300, easing = FastOutSlowInEasing),
        targetOffsetX = { fullWidth -> -fullWidth / 3 }
    ) + fadeOut(
        animationSpec = tween(300, easing = FastOutSlowInEasing)
    )
}

// 返回上一页
popEnterTransition = {
    slideInHorizontally(
        animationSpec = tween(300, easing = FastOutSlowInEasing),
        initialOffsetX = { fullWidth -> -fullWidth / 3 }
    ) + fadeIn(
        animationSpec = tween(300, easing = FastOutSlowInEasing)
    )
}
popExitTransition = {
    slideOutHorizontally(
        animationSpec = tween(300, easing = FastOutSlowInEasing),
        targetOffsetX = { fullWidth -> fullWidth }
    ) + fadeOut(
        animationSpec = tween(300, easing = FastOutSlowInEasing)
    )
}
```

### 功能 75：首次页面切换卡顿优化

**功能描述**：解决应用启动后第一次使用底部导航栏切换页面时出现的卡顿问题，确保首次切换也能流畅响应。

**问题描述**：
- 应用启动后，无论等待多久，第一次用底栏切换页面必卡
- 后续切换流畅，只有第一次卡顿

**问题原因分析**：

**1. Compose 首帧渲染开销**
- 第一次切换到新页面时，Compose 需要进行首次组合（Composition）
- 首次组合需要创建所有 Composable 函数、解析类型、创建布局节点等
- 后续切换可以复用已创建的组件，因此更流畅

**2. ViewModel 首次初始化**
- 每个页面的 ViewModel 在首次访问时才初始化
- ViewModel 初始化涉及依赖注入、数据库查询等
- MainViewModel、ScheduleViewModel 等都有多个 Flow 需要首次订阅

**3. Flow 首次订阅延迟**
- 使用 `SharingStarted.WhileSubscribed(5000)` 策略
- Flow 在首次订阅时才开始收集数据
- 数据库查询、DataStore 读取等都在首次切换时触发

**4. ComposeWarmup 预热不充分**
- 当前预热只覆盖了基础组件
- 没有预热应用特定的组件（如课程卡片、课表格子等）
- 没有预热 ViewModel 和 Flow 订阅

**解决方案**：

**1. 增强预加载策略**
- 在 Application 启动时预初始化所有 ViewModel
- 预订阅关键数据 Flow，使用 `SharingStarted.Lazily` 或 `shareIn`
- 预加载所有页面的初始数据

**2. 优化 Flow 订阅策略**
- 将关键数据的 `SharingStarted.WhileSubscribed(5000)` 改为 `SharingStarted.Lazily`
- 使用 `shareIn` 共享数据流，避免重复订阅
- 在 Application 中预热关键数据

**3. 增强组件预热**
- 扩展 ComposeWarmup，预热应用特定组件
- 预热课程卡片、课表格子、设置项等
- 预热动画组件和过渡效果

**4. 页面预加载**
- 在启动时预加载所有底部导航页面
- 使用 `LaunchedEffect` 在后台预组合页面
- 利用 Compose 的组合缓存机制

**技术实现**：

**1. 修改 DuoScheduleApp.kt**
- 在 Application 启动时预初始化关键数据
- 预订阅所有页面的关键 Flow

**2. 修改 ViewModel**
- 将 `SharingStarted.WhileSubscribed(5000)` 改为 `SharingStarted.Lazily`
- 确保数据在应用启动时就准备好

**3. 增强 ComposeWarmup**
- 添加应用特定组件的预热
- 预热课程卡片、课表格子等

**4. 优化 NavHost**
- 考虑使用 `AnimatedContent` 替代页面切换动画
- 减少首次切换时的动画开销

### 功能 78：上课中通知倒计时修复

**功能描述**：修复上课中通知倒计时显示不准确的问题，确保倒计时与实际课程时间精确同步。

**问题描述**：
- 上课中通知的倒计时比实际课程时间提前约1分钟
- 例如：用户已上课20分钟，但倒计时显示已上课21分钟
- 导致倒计时提前结束，用户还没下课通知就消失了

**问题原因**：

**1. 课程结束时间计算方式不一致**
- `LiveUpdateService` 使用 `courseStartTime.plusMinutes(course.duration)` 计算结束时间
- 但 `duration` 是一个计算属性，可能与课程实际的 `endHour/endMinute` 存在微小差异
- 导致倒计时基于错误的结束时间计算

**2. 通知倒计时结束时间计算精度问题**
- `PromotedNotificationBuilder` 使用 `remainingMinutes * 60 * 1000L` 计算倒计时结束时间
- `remainingMinutes` 通过 `Duration.between().toMinutes()` 获取，会向下取整
- 导致倒计时器基于已取整的分钟数计算，产生误差

**解决方案**：

**1. 统一使用课程的 endHour/endMinute 作为结束时间**
- `LiveUpdateService.refreshCourseState()` 直接使用 `LocalTime.of(course.endHour, course.endMinute)`
- 不再依赖 `duration` 计算属性

**2. 传递精确的课程结束时间**
- `CourseNotificationManager.scheduleOngoingCourseAlarm()` 传递 `endHour` 和 `endMinute`
- `OngoingCourseReceiver` 接收并传递结束时间
- `LiveUpdateService.start()` 接收结束时间参数

**3. 使用精确时间计算倒计时结束时间戳**
- `PromotedNotificationBuilder.buildOngoingNotification()` 接收 `courseEndTime` 参数
- 使用 `Duration.between(now, courseEndTime).toMillis()` 计算精确的毫秒数
- 确保倒计时器与实际课程结束时间精确同步

**技术实现**：

**1. 修改 LiveUpdateService.kt**
- `refreshCourseState()` 使用 `LocalTime.of(course.endHour, course.endMinute)`
- 添加 `currentCourseEndTime` 变量存储结束时间
- `start()` 方法添加 `endHour` 和 `endMinute` 参数
- `onStartCommand()` 读取并设置 `currentCourseEndTime`

**2. 修改 OngoingCourseReceiver.kt**
- 添加 `EXTRA_END_HOUR` 和 `EXTRA_END_MINUTE` 常量
- 读取并传递结束时间到 `LiveUpdateService`

**3. 修改 CourseNotificationManager.kt**
- `scheduleOngoingCourseAlarm()` 传递 `course.endHour` 和 `course.endMinute`

**4. 修改 PromotedNotificationBuilder.kt**
- `buildOngoingNotification()` 添加 `courseEndTime` 参数
- 使用精确时间计算 `endTime` 时间戳

**影响文件**：
- `LiveUpdateService.kt` - 使用精确结束时间
- `OngoingCourseReceiver.kt` - 传递结束时间
- `CourseNotificationManager.kt` - 传递结束时间
- `PromotedNotificationBuilder.kt` - 使用精确时间计算倒计时

**优先级**：高

**状态**：已实现

### 功能 80：小米负一屏小组件适配优化

**功能描述**：优化小米设备负一屏（桌面助手）小组件的显示，解决"载入窗口小部件出现问题"的问题，确保小组件能在小米负一屏正常显示和使用。

**输入**：
- 小米设备系统信息
- 小组件事件状态

**输出**：
- 负一屏可见的小组件
- 小组件状态同步

**业务规则**：

**1. 问题原因分析**
- 小米负一屏（智能助理/桌面助手）对第三方小组件有特殊适配要求
- 仅添加 `miuiWidget: true` 配置不足以让小组件在负一屏显示
- 需要通过 `updateAppWidgetOptions` 设置小组件状态码（miuiWidgetEventCode）

**2. MIUI Widget 状态设置**
- 在小组件更新时，调用 `updateAppWidgetOptions` 设置状态
- 状态码类型：
  - `info1`：课程信息小组件（今日课程、我的课程、Ta的课程）
  - `state1`：功能状态小组件（空闲时间）
  - `other`：其他状态
- 同时设置时间戳 `miuiWidgetTimestamp`

**3. 小组件配置要求**
- XML 配置中必须包含：
  - `miuiWidget: true` - 声明支持 MIUI Widget
  - `miuiWidgetRefresh: exposure` - 曝光刷新模式
  - `miuiWidgetRefreshMinInterval: 60000` - 最小刷新间隔 60 秒
  - `vivo_widget: true` - 同时支持 vivo 设备
- `widgetCategory` 包含 `home_screen|keyguard`

**4. 布局要求**
- 小组件根布局必须使用 RemoteViews 支持的布局类型
- 支持的布局：`FrameLayout`、`LinearLayout`、`RelativeLayout`、`GridLayout`
- 不支持：`ConstraintLayout`（会导致"载入窗口小部件出现问题"）

**5. 错误处理**
- 小组件加载失败时显示"加载中..."状态
- 添加详细日志记录，便于问题排查
- 实现 `onEnabled` 和 `onDisabled` 生命周期回调

**技术实现**：

**1. 新增 MiuiWidgetHelper 工具类**
- `isMiuiWidgetSupported()` - 检测是否支持 MIUI Widget
- `isMiuiWidgetDetailPageSupported()` - 检测是否支持详情页
- `updateMiuiWidgetStatusBeforeUpdate()` - 更新小组件状态

**2. 修改所有小组件 Receiver**
- `ScheduleWidgetReceiverMIUI` - 添加状态设置和错误处理
- `FreeTimeWidgetReceiver` - 添加状态设置
- `MyCoursesWidgetReceiver` - 添加状态设置
- `TaCoursesWidgetReceiver` - 添加状态设置

**3. 状态设置调用顺序**
```kotlin
// 1. 先调用 updateAppWidgetOptions
MiuiWidgetHelper.updateMiuiWidgetStatusBeforeUpdate(
    appWidgetManager,
    appWidgetId,
    MiuiWidgetHelper.EVENT_CODE_COURSE_INFO
)

// 2. 再调用 updateAppWidget
appWidgetManager.updateAppWidget(appWidgetId, views)
```

**影响文件**：
- 新增 `MiuiWidgetHelper.kt` - MIUI Widget 工具类
- `ScheduleWidgetReceiverMIUI.kt` - 添加状态设置和错误处理
- `FreeTimeWidgetReceiver.kt` - 添加状态设置
- `MyCoursesWidgetReceiver.kt` - 添加状态设置
- `TaCoursesWidgetReceiver.kt` - 添加状态设置

- [小米澎湃OS小部件设计规范](https://dev.mi.com/distribute/doc/details?pId=1664)

## AI增强功能

本应用不需要 AI 增强功能，原因如下：

1. **核心功能简单明确**：课程表管理的核心功能是数据的增删改查和展示，不需要复杂的 AI 推理或生成能力。

2. **数据隐私考虑**：应用定位为本地存储，不依赖网络，用户的课程数据完全保存在本地设备上。引入 AI 功能可能需要将数据上传到云端处理，与产品的隐私保护理念冲突。

3. **用户群体特点**：目标用户为大学生情侣，他们更关注的是简单、快速地了解对方的课程状态，而不是 AI 辅助决策。

4. **功能已足够完善**：现有功能已能很好地解决用户痛点（了解对方上课状态、找到共同空闲时间），无需 AI 增强来提升体验。

5. **维护成本考虑**：引入 AI 功能会增加应用的复杂度和维护成本，而收益有限。
