# UI 重构（产品 + 设计 + 前端交付）

## 目标

在理解用户业务与现有代码的基础上，重构项目的前端界面与交互体验：
- 若用户明确“重构哪一部分”，只重构指定范围
- 否则重构整个前端（核心页面与全局组件）

交付物包含：深入 PRFAQ（约 1000 字）、完整需求清单、结构化 PRD（MoSCoW 优先级）、信息架构/任务流、视觉与交互方案、设计系统 Tokens、关键页面线框描述，以及已落地到代码的前端重构实现。

## 执行原则（必须遵守）

- 中文沟通；不使用 emoji
- 不增加空白占位：需要图片时必须使用真实可访问链接（示例见 references）
- 不引入项目里不存在的前端框架/构建链；尽量复用现有依赖与组件模式
- 语义化 HTML5：header/main/aside/section/nav/footer；标题层级正确
- 必备 meta：viewport、color-scheme
- 字体：通过 CDN 引入（如 Google Fonts 的 Inter），并提供中英文混排备用字体栈
- 图标：优先内联 SVG；也可使用 Font Awesome CDN（若需要）
- 响应式：至少三断点；可点击区域 ≥ 44×44px；表格/图表有小屏策略
- 动效：200–300ms 的入场/勾勒/滚动反馈；支持 prefers-reduced-motion 静态回退
- 可访问性：可见焦点、键盘可用、对比度达标、ARIA 语义适度且准确
- “转译而非模仿”：只借鉴气质与方法，不复刻具体作品的构图/配色/字体组合/标语/品牌元素

## 工作流（按顺序执行）

### 1) 范围判断（Scope）

从用户输入中提取重构范围：
- 若出现明确目标（例如“重构首页/导航/表格/登录页/仪表盘/某组件名”），范围锁定到这些页面/组件
- 若未明确，默认全量前端重构：全局布局/导航/页脚/主页面与核心业务路径页面

然后在代码库中确认真实边界：
- 识别前端技术栈（静态站 / React / Vue / Next / Vite 等）与入口文件
- 找到页面路由、布局组件、设计系统/样式方案（CSS Modules / Tailwind / styled-components 等）
- 以“最小破坏”方式改造：优先复用现有结构，只在必要处做架构整理

### 2) PRFAQ（逆向工作法，约 1000 字）

输出 PRFAQ，必须包含：
- Press Release：产品是什么、解决什么问题、核心价值与差异化
- FAQ：至少覆盖目标用户、关键场景、为何现在做、成功指标、边界与不做什么、风险与对策、可访问性与性能策略

### 3) 需求洞察与需求清单

穿透用户表述，补齐显性/隐性需求，形成清单（可按表格输出）：
- 用户角色（至少 3 类）与权限差异
- 使用场景与核心任务路径（Happy path + 关键分支）
- 功能列表、内容策略（文案语气、信息密度、数据默认值）
- 边界与异常：加载/空态/错误/无权限/网络不佳
- 数据结构：页面需要哪些字段、单位/精度、排序/过滤/分页约束（与现有后端对齐）

### 4) PRD（结构化）

输出 PRD，至少包含：
- 目标用户与目标（含非目标）
- 需求列表与优先级（MoSCoW）
- 信息架构：站点地图（Sitemap）与核心任务流（Task Flow）
- 交互说明：关键组件状态（默认/悬停/激活/禁用/错误/空态/加载）
- 指标与埋点建议（如适用）
- 约束：技术、内容、合规、无障碍

### 5) 设计灵感采样（随机 2 位）

从灵感来源池随机采样 2 位，并分别给出：
- “灵感 → 网页实现”的转译：版式/色彩/形态/动态/语义各至少 1 条
- 明确“不复刻”声明：避免具体构图/配色/字体组合/标语/品牌元素

可使用 scripts/ 中的采样脚本；灵感池见 references/。

### 6) 视觉与交互交付方案

基于前端设计原则输出：
- 页面结构（每页的区块与信息层级）
- 组件清单（导航、卡片、表格、图表区、表单、提示与反馈）
- 响应式断点策略与各断点布局变化
- 可访问性规范（对比度、焦点、键盘路径、ARIA、动效回退）
- 动效规范（时长/缓动/触发条件/减少动效回退）

### 7) 设计系统（Design Tokens）

给出 tokens（可以 JSON / CSS Variables 两种形式之一），覆盖：
- 色彩：背景/前景/品牌色/语义色（成功/警告/危险）/边框/阴影
- 字体：字体栈、字号刻度、字重、行高、字距、等宽数字
- 栅格与间距：列数、gutter、spacing scale、圆角、阴影、描边
- 组件密度：默认与紧凑两档（若适用）

同时用文字描述 2–3 个关键页面的线框（信息结构与交互，而非像素级）。

### 8) 前端重构落地（代码交付）

执行步骤：
- 在代码库中定位需要改造的页面/组件与样式入口
- 先做“结构重排”再做“视觉增强”：确保信息架构清晰、可读性与可用性达标
- 抽象可复用组件：卡片、表格、图表区、导航、表单（渐进增强）
- 引入字体与图标 CDN（遵循现有项目方式；静态项目写入 HTML，框架项目写入入口）
- 图片使用真实链接（见 references）；为每张图片提供 alt
- 三断点适配与触控友好尺寸；为表格/图表提供小屏策略（横向滚动、卡片化、折叠列等）
- 统一状态：加载/空/错误/成功反馈；支持键盘与焦点可见
- 按需加动效，并实现 prefers-reduced-motion 回退

验收与自检：
- 视觉：对比度达标、排版秩序清晰、信息密度合理
- 交互：键盘可用、焦点可见、点击热区达标
- 响应式：三断点通过；窄屏无溢出；表格/图表可用
- 性能：避免无意义大图；可延迟加载非首屏资源（若项目已有方案）
